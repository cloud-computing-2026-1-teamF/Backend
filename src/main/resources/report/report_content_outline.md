# 분석 이력 기반 PDF 보고서 — 내용 구성안

회의록(5/28)에서 합의된 방향을 보고서 지면으로 옮긴 구성안. 각 섹션은 `output.schema.json`
필드와 1:1로 대응하고, `make_sample_pdf.py` 가 이 구조 그대로 렌더한다.

## 설계 원칙 (회의록 근거)

- **AI 점수 확보용 도입**: 정량 지표(점수·유동인구)를 일반인이 이해하기 어려우니 LLM이 줄글로 풀어
  'AI를 효과적으로 활용'했음을 심사에 어필. (배점: 공공데이터 활용 + AI 혁신성 45점)
- **콘텐츠 두 축**: ① 전문가 줄글 총평, ② ROI/매출 기반 액션 가이드.
- **생존율 수치화 금지**: "생존율 80%" 같은 임의 수치는 근거 반박에 취약 → 검증 가능한 **예상매출**
  기준 ROI/BEP만 제시(본죽 본사 상권분석 벤치마킹).
- **경쟁점 분석**: 네이버 리뷰 본문(스크래퍼)에서 맥락을 뽑아 "어떤 컨셉이 유리한가"로 연결.
- **Freemium**: 맛보기 무료 + 심화 유료.
- **투 트랙 데모**: 실시간 생성 + 사전 생성 샘플 PDF 폴백.

## 표지 / 헤더

매물 주소 · 업종 · 생성일 · **입지 점수 + 등급** · 한 줄 헤드라인(`headline`).

## 섹션 구성

| # | 섹션 | 내용 | 회의록 근거 | 데이터 출처 | output 필드 |
|---|---|---|---|---|---|
| 01 | 전문가 총평 | 입지 성격 → 누구를 노려야 하는가, 줄글 3~5문단 | "AI가 사람처럼 풀어 설명" | score·foot·comp·rev | `summary` |
| 02 | 핵심 지표 | 추정 월매출 vs 지역평균, 점수, 회수기간 KPI | 정량 지표 시각화 | market | `estimatedRevenue` |
| 03 | 수익성(ROI·BEP) | 초기투자금/순이익/회수개월 + "피크 N잔" 액션선 | "ROI·피크 판매량 액션 가이드" | derived(백엔드 계산) | `roi`,`bep` |
| 04 | 추천 창업 방향 | 컨셉/메뉴/가격대/인테리어/타깃 | "어떤 카페를 차리면 좋을까" | competitors 리뷰 + 업종 | `recommendation` |
| 05 | 주변 경쟁점 분석 | 경쟁 강도 + 리뷰에서 추출한 기회/주의 신호 | "리뷰 NLP 맥락 분석" | competitors.reviewSnippets | `competition` |
| 06 | 리스크 · 기회 | 정성 항목 각 2~4개 | 생존율 대신 정성 평가 | 종합 | `risks`,`opportunities` |
| — | 면책 | 추정치·가정값·데이터 한계 명시 | 신뢰도 방어 | assumptions | `disclaimers` |

## Freemium 게이팅 (회의록: 맛보기 → 심화 유료)

- **FREE**: 01·02 + 총평 1문단 미리보기, 03~06은 블러 + "PRO에서 전체 확인" 워터마크.
- **PRO/BUSINESS**: 전체 + PDF 다운로드. (백엔드 `validateReportAccess(tier)` 게이트)

## B2B / B2G 변형 (회의록 확장안 — 같은 데이터, 다른 지면)

- **중개업소용 1장 요약**: 점수 + 추정매출 + ROI 한눈 카드. 건당 과금. (매물 브리핑·영업용)
- **지자체용 정책 보고서**: 자치구 단위 쇠퇴·임대료·고령화 집계. 라이선스 판매.

## 렌더 가이드

- A4 세로, PRO 2~3p / FREE 미리보기 1p.
- **라이브**: Kotlin `openhtmltopdf` (한글 폰트 임베드 권장 — Noto Sans KR / Nanum).
- **데모 폴백**: `make_sample_pdf.py`(reportlab). 현재 Adobe-Korea1 CID 사용 → Chrome/Acrobat에선
  정상, 일부 리눅스 뷰어에선 글자 간격 이슈. 배포 시 TTF 임베드로 전환 권장.
